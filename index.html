Painel Estratégico do Projeto

🎯 O que é este projeto

▼

A Central Movelmar é o sistema que centraliza toda a operação da representação comercial de móveis planejados Movelmar. Serve para o Rica (representante) gerenciar seus 11 lojistas ativos, manter catálogos atualizados de cores e ferragens, registrar demandas/assistências técnicas e automatizar comunicações com a fábrica. O problema que resolve: sair de planilhas, WhatsApp e informação espalhada — tudo num lugar só, acessível pelo celular, com dados em tempo real.

📝 Decisões Técnicas

▼

Por que prefixo mvmr_ vs cncta_?

mvmr_ = exclusivo Movelmar (cores, ferragens, contatos fábrica). cncta_ = tabelas que podem servir outras representações no futuro (empresas, demandas, mensagens). Separação permite escalar sem refatorar.

Por que RLS público em mvmr_cores?

O catálogo de cores é compartilhado com lojistas via link público (GitHub Pages). A anon key do Supabase permite leitura sem login. Dados não são sensíveis — são catálogo comercial.

Por que ASS-XXX para assistências?

Formato curto, memorável, fácil de referenciar no WhatsApp. Trigger automático no banco garante unicidade e sequência. "ASS" = assistência. Começa em 001.

Por que campo textura nas cores?

Mesma cor pode ter variantes de acabamento (matt = fosco, micro = microporo). Cinza Cobalto Berneck, por exemplo, tem duas linhas. Campo diferencia sem duplicar nome.

Por que GitHub Pages e não Vercel/Netlify?

Simplicidade máxima. HTML estático, sem build, sem framework. Push = deploy. Rica pode editar direto no GitHub se precisar. Zero custo.
